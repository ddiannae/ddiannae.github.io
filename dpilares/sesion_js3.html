Scope o alcance

El scope determina a accesibilidad o visibilidad de las variables. Hay dos tipos:

  • Scope global
  • Scope local

En JS cada función crea su propio scope. Las variables que declaramos dentro de una función solo pueden accederse dentro de ella.

Scope local

Las variables dentro de una función son locales a ella. Incluyendo los parámetros. 

          
                    //Aquí no podemos utilizar la variable parrafo
                    function leerParrafo() {
                        let parrafo = "";
                        //Aquí podemos utilizar la variable parrafo
                    }

Podemos declarar variables con el mismo nombre en diferentes funciones porque se crean cuando la función inicia y se borran cuando se completa.

Scope global

Si una variable se crea fuera de una función, se vuelve global y todas las funciones pueden acceder a ella. 

          
                    let parrafo = "";
                    //Aquí podemos utilizar la variable parrafo
                    function leerParrafo() {
                        //Aquí también podemos utilizar la variable parrafo
                    }

Si asignamos valor a una variable sin declararla, tambié se vuelve global

  • Los objetos y las funciones también son variables.
  • El scope global de JS es todo el ambiente.
  • La vida de las variables inicia cuando se declaran.
  • Las variables locales se borran cuando la función se completa.
  • Las variables globales las limpia nuestro navegador cuando cerramos la pestaña o la ventana.

Hoisting

Las declaración de variables con var se mueve por default al inicio del scope. Se pueden usar variables antes de declararlas pero la iniciación no se mueve.

Son lo mismo:

          
                    x = 3;
                    console.log(x);
                    var x;
                    
          
                        var x;
                        x = 3;
                        console.log(x);

Con let y const el scope sabe que va a usar las variables pero no lo permite. Con let causa un ReferenceError y con const un error de sintaxis.

Las declaraciones de funciones sí se hacen hoist.

Para evitar confusiones declara las variables locales al inicio de la función y las globales al inicio del script.

Arrow functions (¿funciones flecha?)

Se crearon para simplificar el código.

          
                    // Función original. De hecho es una función anónima
                    function (a){
                      return a + 100;
                    }
                    // 1. Quitamos la palabra "function" y ponemos una flecha entre los argumentos y
                    // la llave de unicio
                    (a) => {
                      return a + 100;
                    }
                    
                    // 2. Quitamos las llaves y el "return". Solo para código con una sola línea.
                    (a) => a + 100;
                    
                    // 3. Quitamos los paréntesis del argumento
                    a => a + 100;

Con más líneas y nombre

          
                    // Función tradicional
                    function (a, b){
                        let chuck = 42;
                        return a + b + chuck;
                    }

                    // Función flecha
                    (a, b) => {
                        let chuck = 42;
                        return a + b + chuck;
                    }

                    // Función tradicional
                    function bob (a){
                        return a + 100;
                    }

                    // Función flecha
                    let bob = a => a + 100;

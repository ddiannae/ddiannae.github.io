En general, un programa es una lista de instrucciones ejecutadas por la computadora. En un lenguaje de programación, las instrucciones se llaman declaraciones.

Las declaraciones en JS se componen de: valores, operadores, expresiones, palabras reservadas y comentarios.

Sintaxis: Las reglas que componen un lenguaje. Describen las declaraciones válidas.

Valores

Hay dos tipos:

  • Literales. Valores fijos

                        /* Números*/
                        /* Por cierto, esto es un comentario multi-línea 
                        en JavaScript */
                        10
                        8.340
                        /* Strings */
                        /* Los comentarios no ejecutan nada. Sirven para que, 
                        como desarrolladoras,  ayudemos a nuestra yo del 
                        futuro o a alguien más. */
                        "Hola, con comillas dobles"
                        'Hola, con comillas simples'
  • Literales. Valores fijos
  • Variables. Guardan datos que pueden cambiar

                        let a, b; // Declaración
                        a = 10; // Asignación
                        b = 35;
                        let multiplicacionDeDosVariables = a * b; //esto es camelCase:
                        // utilizar mayúsculas en las primeras letras de nombres
                        // compuestos de variables y funciones. Primera minúscula.
                        // Esto es un comentario de una sola línea.

¿Qué identificadores de variables son válidos?

Antes de 2015, para declarar variables sólamente se usaba var, ahora se utiliza let y const

Una de las diferencias entre var y let es su scope, es decir, hasta dónde están disponibles las variables para ser usadas.

Por ahora: utilizar const si es una variable que no va a modificarse, y let en otros casos.

Explicación: Inglés y Español

Declarando variables con var

Cosa que no debemos hacer


                        var unaVariable; 
                        console.log(unaVariable); // ¡Un mensaje en la consola!
                        unaVariable = "Hola, soy una variable";
                        console.log(unaVariable);
                        unaVariable = "Ahora tengo un nuevo valor";
                        console.log(unaVariable);
                        /* Mucho código que no tiene nada
                        que ver con la variable unaVariable y 
                        ya se nos olvidó que la habíamos declarado */
                        var unaVariable; // Redeclaración
                        console.log(unaVariable); // ¿Qué valor tiene?

¡Guarda su valor después de ser redeclarada!

Declarando variables con let

¿Podemos hacer lo mismo?


                        let unaVariable; 
                        console.log(unaVariable); //  Mensaje en la consola!
                        unaVariable = "Hola, soy una variable";
                        console.log(unaVariable);
                        unaVariable = "Ahora tengo un nuevo valor";
                        console.log(unaVariable);
                        /* Mucho código que no tiene nada
                        que ver con la variable unaVariable y 
                        ya se nos olvidó que la habíamos declarado */
                        let unaVariable; // Redeclaración

Uncaught SyntaxError: Identifier 'unaVariable' has already been declared

¿Y qué pasa con const?


                        console.log("Intentemos declarar y luego asignar una variable");
                        const unaVariableConValorFijo; // Declaramos
                        unaVariableConValorFijo = "No cambiaré mi valor"; // Asignamos

Uncaught SyntaxError: Missing initializer in const declaration

Si utilizamos const, tenemos que inicializar la variable

¿Y qué pasa con const?


                        console.log("¿Y si le tratamos de cambiar el valor después?");
                        const unaVariableConValorFijo = "No cambiaré mi valor"; 
                        unaVariableConValorFijo = "Bueno, mejor sí";

Uncaught TypeError: Assignment to constant variable.

Por eso es para valores constantes.

¿Observaron alguna diferencia en los mensajes en la consola?

  • SyntaxError: Nada del código se ejecuta.
  • Otros errores como TypeError: Termina la ejecución al momento de encontrar el error.

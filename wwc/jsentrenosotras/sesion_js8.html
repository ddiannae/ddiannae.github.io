JS entre nosotras

Funciones

Las funciones son como los ladrillos en programación.

Nos permiten definir código una vez y ejecutarlo muchas veces. Reutilizar código.

Pueden tomar valores, hacer cálculos y generar el resultado correspondiente.

Definición


                        function productoDosNumeros(num1, num2) {
                            return num1*num2;
                        }
  • Palabra reservada function.
  • Un nombre + paréntesis.
  • Parámetros opcionales separados por comas.
  • Código a ejecutarse entre {}.
  • La expresió de return concluye la ejecución.

Invocación


                        let x = 8;
                        let z = productoDosNumeros(x, 3);
  • El código se ejecuta hasta que se la función se invoca.
  • Al invocar se pasan los valores de las variables.
  • Las variables dentro de la función, son locales.

Ejercicio

Expresión de función (function expression)

Declarar una variable como función.


                       const saludar = function() {
                           console.log("buen día!");
                       };

Diferencia: no hay hoisting en estas declaraciones.

Parámetros y argumentos


                        const saludar = function(nombre) {
                            /* nombre es un parámetro, una variable
                            que solo va a existir dentro de esta función, 
                            una variable local*/
                            console.log(`buen día, ${nombre}!`);
                        };

                        saludar("chicas");
                        /* "chicas" es un argumento, es el valor que va a 
                        tomar el parámetro nombre en la función */

El orden de los parámetros importa


                        const saludar = function(dia, nombre) {
                            console.log(`buen ${dia}, ${nombre}!`);
                        };
                        /*No son lo mismo*/
                        saludar("jueves","chicas");
                        saludar("chicas", "jueves");

¿Qué pasa si mandamos llamar la función sin ningún argumento? 


                        saludar();

Para evitar eso podemos declarar valores default para los parámetros


                        const saludar = function(dia = "día", nombre = "tú") {
                            console.log(`buen ${dia}, ${nombre}!`);
                        };
                        saludar();
                        saludar("jueves","chicas");

Valores de retorno, return

Las funciones pueden regresar valores, que almacenamos en otras variables.


                        const celsiusAFahrenheit = function (grados) {
                            return (grados-32)*(5/9)
                        };

                        const gradosF = celsiusAFahrenheit(90);

Funciones flecha

Se crearon para simplificar el código.

          
                    // Función original.
                    const sumacien = function (a) {
                      return a + 100;
                    }
                    // 1. Quitamos la palabra "function" y ponemos una flecha
                    //  entre los argumentos y la llave de unicio
                    const sumacien = (a) => {
                      return a + 100;
                    }
                    
                    // 2. Quitamos las llaves y el "return". Solo para 
                    // código con una sola línea.
                    const sumacien = (a) => a + 100;
                    
                    // 3. Quitamos los paréntesis del parámetro si 
                    // solo es uno
                    const sumacien = a => a + 100;

Funciones y métodos

Los métodos son también funciones, pero están asociadas a los objetos y los llamamos con la notación de punto. 


                        const texto = "Lorem ipsum dolor sit amet, consectetur adipiscing elit. 
                        Donec ut pretium libero, sit amet mollis."
                        texto.indexOf("adipiscing");
                        
                        const arr = [1, 5, 6, 2];
                        arr.pop();

Funciones callback

Son funciones que se mandan llamar dentro de otras.


                        const carritoCompra = ["leche", "queso", "pan", "jamón"];
                        carritoCompra.forEach(function (producto) {
                            console.log(producto)
                        })

Podemos también declararlas a parte y pasar su declaración.


                        const carritoCompra = ["leche", "queso", "pan", "jamón"];
                        const muestraProducto = producto => console.log(producto);
                        carritoCompra.forEach(muestraProducto);

Scope o alcance

El scope determina a accesibilidad o visibilidad de las variables. Hay dos tipos:

  • Scope global
  • Scope local

En JS cada función crea su propio scope. Las variables que declaramos dentro de una función solo pueden accederse dentro de ella.

Scope local

Las variables dentro de una función son locales a ella. Incluyendo los parámetros. 

          
                    //Aquí no podemos utilizar la variable parrafo
                    function leerParrafo() {
                        let parrafo = "";
                        //Aquí podemos utilizar la variable parrafo
                    }

Podemos declarar variables con el mismo nombre en diferentes funciones porque se crean cuando la función inicia y se borran cuando se completa.

Scope global

Si una variable se crea fuera de una función, se vuelve global y todas las funciones pueden acceder a ella. 

          
                    let parrafo = "";
                    //Aquí podemos utilizar la variable parrafo
                    function leerParrafo() {
                        //Aquí también podemos utilizar la variable parrafo
                    }

Si asignamos valor a una variable sin declararla, tambié se vuelve global

  • Los objetos y las funciones también son variables.
  • El scope global de JS es todo el ambiente.
  • La vida de las variables inicia cuando se declaran.
  • Las variables locales se borran cuando la función se completa.
  • Las variables globales las limpia nuestro navegador cuando cerramos la pestaña o la ventana.
